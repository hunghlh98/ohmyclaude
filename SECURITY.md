# Security Policy

## Supported Versions

ohmyclaude follows Semantic Versioning. Only the latest minor version receives security fixes.

| Version | Supported |
|---------|-----------|
| 1.x     | Yes       |
| 0.x     | No — please upgrade (see `MIGRATION.md` when published) |

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public issue**.

Email: **hunghlh@vng.com.vn**
Subject prefix: `[ohmyclaude security]`

Include:
1. A description of the issue and its impact.
2. Steps to reproduce (minimum viable case).
3. The ohmyclaude version affected (`node scripts/validate.js` prints it).
4. Your contact info for follow-up.

## Triage SLA

| Severity | Acknowledgement | Target fix |
|----------|-----------------|------------|
| Critical (remote code execution, secret exfiltration, supply-chain compromise) | 48 hours | 7 days |
| High (privilege escalation, data exposure) | 72 hours | 30 days |
| Medium (DoS, misconfiguration risk) | 1 week | 60 days |
| Low (hardening, defense-in-depth) | 2 weeks | Best effort |

## Scope

**In scope** — shipped plugin surface:
- Agent prompts in `agents/`
- Skills in `skills/`
- Commands in `commands/`
- Hook scripts in `hooks/scripts/`
- Rules in `rules/`
- Install manifests in `manifests/`
- Plugin manifest in `.claude-plugin/`
- Build/validation scripts in `scripts/`

**Out of scope** — not ohmyclaude:
- The Claude Code CLI itself (report to Anthropic).
- Third-party MCP servers configured by the end user — report to their maintainers.
- End-user code generated by agents (review it yourself before running).

## Coordinated Disclosure

We ask reporters to allow 90 days from acknowledgement before public disclosure, or until a fix is released — whichever comes first. We will credit the reporter in the release notes unless you request otherwise.

## Non-Goals

ohmyclaude is an orchestration layer. It cannot guarantee the safety of:
- Code produced by agents (agents can introduce vulnerabilities; `@sam-sec` and `@stan-standards` are first-line review, not a substitute for human security review).
- External tools invoked through Bash or MCP.
- User-installed third-party skills or agents added on top of ohmyclaude.

Always run `node scripts/validate.js` after modifying plugin content, and review agent-generated code before committing.
